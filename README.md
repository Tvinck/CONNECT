# CONNECT — запуск на своём компьютере

Внутренняя платформа команды BAZZAR Group (Next.js 14 + Supabase).

Ниже — полная инструкция, как скачать проект и запустить его локально.
Шаги 1–4 делаются **один раз**. Дальше — только «Запуск» и «Обновление».

---

## Что должно быть установлено (один раз)

### 1. Node.js
Скачай **LTS-версию** с https://nodejs.org → установи (всё «Далее → Далее»).
Проверка — открой терминал и набери:
```bash
node -v
npm -v
```
Если показались номера версий (например `v20.x.x`) — всё ок.

### 2. Git
- **Windows:** https://git-scm.com/download/win → установи.
- **macOS:** уже есть; если нет — установится при первом вызове `git`.

Проще всего для новичка — **GitHub Desktop**: https://desktop.github.com
Он скачивает проект и обновляет его в пару кликов, без команд.

## 3. Вставить ключи Supabase (один раз)

В корне проекта создай файл с именем **`.env.local`** и вставь в него:

```env
NEXT_PUBLIC_SUPABASE_URL=https://<your-project-ref>.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=<вставь_свой_anon_key_из_Supabase>
SUPABASE_SERVICE_ROLE_KEY=<вставь_свой_service_role_key_из_Supabase>
```

> ⚠️ Этот файл специально **не хранится в Git** (в нём секреты). Поэтому при
> скачивании его нет — нужно создать вручную один раз. Образец полей —
> в файле `.env.local.example`.
>
> 🔑 **`SUPABASE_SERVICE_ROLE_KEY`** нужен, чтобы CEO мог приглашать сотрудников
> (создаёт им вход). Возьми его в Supabase: **Settings → API → service_role**
> (секция «Project API keys», кнопка «Reveal»). Этот ключ — только для сервера,
> никогда не вставляй его в `NEXT_PUBLIC_*` и не публикуй.

## 3.1 Применить миграцию для чатов (один раз)

Открой Supabase → **SQL Editor** → New query, вставь и выполни содержимое
файла **`supabase/migrations/0002_features.sql`** (добавляет ответы в чате).

## 4. Установить зависимости (один раз)

Открой терминал **в папке проекта** и выполни:
```bash

```
Это скачает все библиотеки (займёт 1–3 минуты).

> Как открыть терминал в папке:
> - **GitHub Desktop:** меню `Repository → Open in Terminal` (или Command Prompt).

> - **VS Code:** открой папку проекта, затем `Terminal → New Terminal`.

## Если что-то не работает

| Симптом | Что делать |
|---|---|
| `command not found: npm` | Node.js не установлен / перезапусти терминал |
| Белый экран, ошибка про `SUPABASE` | Проверь, что создал `.env.local` (шаг 3) и перезапустил `npm run dev` |
| «Неверная почта или пароль» | В Supabase не выполнен `seed.sql` — прогони его в SQL Editor |
| Порт занят (`3000 in use`) | Закрой другой запущенный `npm run dev` или открой `http://localhost:3001` |

Если непонятно — скопируй текст ошибки из терминала и пришли мне.

## 📂 Структура проекта

* **`app/`** — роутинг Next.js App Router (страницы и API).
  * **`api/shop/ggsel/`** — API интеграции с GGSel:
    * `sync-chats` — синхронизация переписки с GGSel.
    * `send-message` — отправка ответа оператора обратно в GGSel.
    * `verify` — проверка кода покупки (возвращает 200).
    * `link` — привязка покупки к подписке.
* **`components/`** — React-компоненты интерфейса.
  * **`support/`** — чат техподдержки в CRM (отображает чаты Connect, GGSel, Telegram).
  * **`projects/`** — управление VPN-серверами, подписками и рефералами.
* **`lib/`** — утилиты и инстансы клиентов (Supabase anon/admin).
* **`scripts/`** — вспомогательные скрипты, тесты API, сиды базы данных и файлы миграций.
* **`operator_bot.js`** — фоновый Telegram-бот для уведомления операторов о новых сообщениях.
* **`pachca_bot.js`** — фоновый бот для отправки Web-Push уведомлений, интеграции с Пачкой и запуска cron-задач (включая ежеминутный опрос GGSel).

## 🤖 Запуск фоновых служб (ботов) на VPS

Для корректной работы уведомлений операторов и синхронизации с GGSel на VPS-сервере должны крутиться два фоновых процесса Node.js.

### 1. Переменные окружения (`.env.local`)
Убедитесь, что на сервере настроены следующие переменные:
```env
TELEGRAM_OPERATOR_CHAT_ID=<id_чата_куда_слать_уведомления>
PACHCA_WEBHOOK_URL=<урл_вебхука_пачки>
GGSEL_SELLER_ID=<id_продавца_ggsel>
GGSEL_API_KEY=<api_ключ_ggsel>
```

### 2. Запуск через PM2 (рекомендуется)
На сервере установите `pm2` для автоматического перезапуска ботов при падении или перезагрузке VPS:
```bash
# Установка PM2 глобально
npm install -g pm2

# Запуск ботов
pm2 start pachca_bot.js --name "connect-cron-bot"
pm2 start operator_bot.js --name "connect-tg-bot"

# Сохранение конфигурации для автозапуска при рестарте ОС
pm2 save
pm2 startup
```

### 3. Логи ботов
```bash
pm2 logs connect-cron-bot
pm2 logs connect-tg-bot
```

## 🛒 Интеграция с GGSel (Двусторонняя синхронизация)

### Схема работы:
1. **Синхронизация входящих**: Раз в минуту `pachca_bot.js` дергает метод `POST /api/shop/ggsel/sync-chats`.
2. **Обход внешних ключей**: База данных Supabase запрещает сохранять сообщения для несуществующих пользователей. Для решения этой проблемы при первом импорте чата с GGSel скрипт автоматически создает неактивную VPN-подписку-заглушку (`vpn_subscriptions`) со статусом `active` и `traffic_limit = 0`.
3. **Отправка исходящих**: При ответе оператора в CRM, клиентская часть определяет, что чат относится к GGSel (по префиксу проекта `GGSel (Заказ ...)` и фейковому UUID), и перенаправляет запрос на `POST /api/shop/ggsel/send-message`. Этот эндпоинт авторизуется в GGSel API и транслирует сообщение покупателю.
